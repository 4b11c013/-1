# 上傳專案到 GitHub 的步驟

## 一、要上傳的內容（你的專案已經包含）

上傳時會上傳**整個專案資料夾**，但以下會被自動排除（已在 `.gitignore` 裡）：

- ❌ **不要上傳**：`node_modules`（依賴套件，別人會自己 `npm install`）
- ❌ **不要上傳**：`dist`（打包後的檔案）
- ❌ **不要上傳**：`.env`、各種 log 檔

**會上傳的檔案與資料夾：**

| 項目 | 說明 |
|------|------|
| `src/` | 所有原始碼（App.jsx、pages、components 等） |
| `public/` | 靜態資源（若有） |
| `index.html` | 入口 HTML |
| `package.json` | 專案依賴與腳本 |
| `package-lock.json` | 鎖定依賴版本 |
| `vite.config.js` | Vite 設定 |
| `tailwind.config.js` | Tailwind 設定 |
| `postcss.config.js` | PostCSS 設定 |
| `.gitignore` | 告訴 Git 哪些不要上傳 |

## 二、上傳步驟

### 方法 A：用指令列（推薦）

1. **安裝 Git**（若尚未安裝）  
   到 https://git-scm.com/ 下載並安裝。

2. **在專案根目錄開啟終端機**  
   路徑應為：`c:\Users\jer93\OneDrive\桌面\版1`

3. **初始化 Git 並第一次提交：**
   ```bash
   git init
   git add .
   git commit -m "首次提交：鼻胃管灌食醫療照護流程指引系統"
   ```

4. **在 GitHub 建立新倉庫：**
   - 打開 https://github.com/new
   - Repository name 可填：`medical-care-system` 或自訂名稱
   - 不要勾選 "Add a README file"（專案已有檔案）
   - 按 **Create repository**

5. **把本機專案連到 GitHub 並上傳：**
   ```bash
   git remote add origin https://github.com/你的帳號/你的倉庫名稱.git
   git branch -M main
   git push -u origin main
   ```
   把 `你的帳號` 和 `你的倉庫名稱` 換成你剛建立的倉庫網址。

---

### 方法 B：用 GitHub Desktop

1. 下載並安裝 [GitHub Desktop](https://desktop.github.com/)
2. 登入你的 GitHub 帳號
3. **File → Add local repository**，選擇資料夾：`c:\Users\jer93\OneDrive\桌面\版1`
   - 若顯示「不是 Git 倉庫」，先選 **create a repository**，路徑選同一個資料夾
4. 左側勾選要提交的檔案，寫 commit 說明，按 **Commit to main**
5. 上方 **Publish repository**，設定倉庫名稱與是否公開，再按 **Publish**

## 三、之後要更新專案時

在專案資料夾裡執行：

```bash
git add .
git commit -m "說明你做了什麼修改"
git push
```

## 四、解決 GitHub Pages 出現 404（重要）

**為什麼會 404？**  
若只上傳原始碼，GitHub Pages 不會幫你執行 Vite 打包，瀏覽器會去請求 `/manifest.json`、`/src/main.jsx` 等，在專案頁網址（例如 `https://使用者名.github.io/倉庫名稱/`）下會變成錯誤路徑而 404。

**做法：用 GitHub Actions 自動建置並部署**

1. 專案已加入 `.github/workflows/deploy-gh-pages.yml`，推送到 `main` 時會自動建置並部署。
2. 在 GitHub 倉庫：**Settings → Pages**：
   - **Source** 選 **GitHub Actions**。
3. 之後每次 `git push` 到 `main`，約 1～2 分鐘後網站會更新。  
   網址為：`https://4b11c013.github.io/你的倉庫名稱/`（專案頁）或 `https://4b11c013.github.io/`（若倉庫名為 `4b11c013.github.io`）。

**專案已做的修正：**
- `vite.config.js` 已設 `base: './'`，資源路徑在專案頁下也會正確。
- 部署的是**建置後的 `dist`**，不是原始碼，所以不會再出現 manifest / main.jsx / zodiak.css 的 404。

## 五、別人要下載你的專案時

```bash
git clone https://github.com/你的帳號/你的倉庫名稱.git
cd 你的倉庫名稱
npm install
npm run dev
```

---

**總結：** 在專案根目錄執行 `git init` → `git add .` → `git commit`，到 GitHub 建立新倉庫後用 `git remote add origin` 和 `git push` 上傳。記得在倉庫 **Settings → Pages** 將 Source 設為 **GitHub Actions**，網站才會正常顯示、不會 404。
