# 上傳專案到 GitHub 的步驟

## 一、要上傳的內容（你的專案已經包含）

上傳時會上傳**整個專案資料夾**，但以下會被自動排除（已在 `.gitignore` 裡）：

- ❌ **不要上傳**：`node_modules`（依賴套件，別人會自己 `npm install`）
- ❌ **不要上傳**：`dist`（打包後的檔案）
- ❌ **不要上傳**：`.env`、各種 log 檔

**會上傳的檔案與資料夾：**

| 項目 | 說明 |
|------|------|
| `src/` | 所有原始碼（App.jsx、pages、components 等） |
| `public/` | 靜態資源（若有） |
| `index.html` | 入口 HTML |
| `package.json` | 專案依賴與腳本 |
| `package-lock.json` | 鎖定依賴版本 |
| `vite.config.js` | Vite 設定 |
| `tailwind.config.js` | Tailwind 設定 |
| `postcss.config.js` | PostCSS 設定 |
| `.gitignore` | 告訴 Git 哪些不要上傳 |

## 二、上傳步驟

### 方法 A：用指令列（推薦）

1. **安裝 Git**（若尚未安裝）  
   到 https://git-scm.com/ 下載並安裝。

2. **在專案根目錄開啟終端機**  
   路徑應為：`c:\Users\jer93\OneDrive\桌面\版1`

3. **初始化 Git 並第一次提交：**
   ```bash
   git init
   git add .
   git commit -m "首次提交：鼻胃管灌食醫療照護流程指引系統"
   ```

4. **在 GitHub 建立新倉庫：**
   - 打開 https://github.com/new
   - Repository name 可填：`medical-care-system` 或自訂名稱
   - 不要勾選 "Add a README file"（專案已有檔案）
   - 按 **Create repository**

5. **把本機專案連到 GitHub 並上傳：**
   ```bash
   git remote add origin https://github.com/你的帳號/你的倉庫名稱.git
   git branch -M main
   git push -u origin main
   ```
   把 `你的帳號` 和 `你的倉庫名稱` 換成你剛建立的倉庫網址。

---

### 方法 B：用 GitHub Desktop

1. 下載並安裝 [GitHub Desktop](https://desktop.github.com/)
2. 登入你的 GitHub 帳號
3. **File → Add local repository**，選擇資料夾：`c:\Users\jer93\OneDrive\桌面\版1`
   - 若顯示「不是 Git 倉庫」，先選 **create a repository**，路徑選同一個資料夾
4. 左側勾選要提交的檔案，寫 commit 說明，按 **Commit to main**
5. 上方 **Publish repository**，設定倉庫名稱與是否公開，再按 **Publish**

---

## 三、之後要更新專案時

在專案資料夾裡執行：

```bash
git add .
git commit -m "說明你做了什麼修改"
git push
```

---

## 四、別人要下載你的專案時

```bash
git clone https://github.com/你的帳號/你的倉庫名稱.git
cd 你的倉庫名稱
npm install
npm run dev
```

---

**總結：** 你只需要在專案根目錄執行 `git init` → `git add .` → `git commit`，再到 GitHub 建立新倉庫，用 `git remote add origin` 和 `git push` 就能完成上傳。`.gitignore` 會自動排除 `node_modules` 等不需上傳的檔案。
